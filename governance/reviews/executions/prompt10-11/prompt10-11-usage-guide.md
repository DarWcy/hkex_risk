# Prompt 10 and 11 Usage Guide

## Overview
This guide provides detailed instructions for using Prompt 10 (Test Execution and Results Validation + Coverage Analysis) and Prompt 11 (BDD Scenario Execution and Reporting + Automation Integration) to ensure comprehensive test execution and BDD scenario validation.

## Prompt 10: Test Execution and Results Validation + Coverage Analysis

### Purpose
Generate comprehensive test execution plans, results validation frameworks, and coverage analysis reports based on the generated test cases and BDD scenarios.

### When to Use
- Test cases and BDD scenarios need to be executed
- Results validation is required
- Coverage analysis is needed to ensure comprehensive testing
- Test automation is desired
- Detailed test execution reports are needed

### Input Requirements
1. **Test Cases**: Paste the complete test case content generated by Prompt 6
2. **BDD Scenarios**: Paste the complete BDD scenario content generated by Prompt 7

### Output Expectations
1. **Test Execution Plan**: Detailed execution plans including test environments, dependencies, and execution sequences
2. **Results Validation Framework**: Validation frameworks to verify test results against expected outcomes
3. **Coverage Analysis Reports**: Comprehensive coverage reports including rule coverage, scenario coverage, and test case effectiveness metrics
4. **Test Automation Scripts**: Automation scripts for test execution and results collection, stored in `tests/automation/` directory
5. **Results Reporting**: Detailed test execution reports with actionable insights and improvement recommendations
6. **Process File**: `PROMPT10-OUTPUT.md` stored in `governance/` directory

### Usage Steps
1. **Preparation**:
   - Gather test cases from Prompt 6 output
   - Gather BDD scenarios from Prompt 7 output
   - Identify test environments and dependencies

2. **Execution**:
   - Run Prompt 10 with prepared inputs
   - Review generated test execution plans
   - Validate results validation frameworks
   - Verify coverage analysis reports
   - Test automation scripts

3. **Review**:
   - Use `governance/reviews/prompt10-review.md` for review tracking
   - Apply `governance/reviews/test-execution-review-template.md` for structured feedback
   - Calculate confidence levels using `governance/reviews/confidence-assessment.md`
   - Perform failure analysis if needed using `governance/reviews/failure-analysis-template.md`

4. **Implementation**:
   - Follow the generated test execution plans
   - Execute test cases and BDD scenarios
   - Collect and validate results
   - Generate coverage analysis reports
   - Document findings and recommendations

5. **Verification**:
   - Verify all test cases are executed
   - Validate results against expected outcomes
   - Confirm coverage analysis completeness
   - Test automation script functionality

### User Type Templates
- **Type A (BA)**: Business-focused test execution templates with business rule verification
- **Type B (QA Lead)**: Quality-focused test execution templates with test case management
- **Type C (Automation Tester)**: Automation-focused test execution templates with CI/CD integration
- **Type D (Mixed/General)**: Universal test execution templates with balanced coverage

## Prompt 11: BDD Scenario Execution and Reporting + Automation Integration

### Purpose
Generate BDD execution frameworks, automated test runners, and comprehensive execution reports based on the generated BDD scenarios.

### When to Use
- BDD scenarios need to be executed
- Automated BDD execution is desired
- Result aggregation across multiple runs is needed
- Detailed BDD execution reports are required
- CI/CD integration for BDD scenarios is needed

### Input Requirements
1. **BDD Scenarios**: Paste the complete BDD scenario content generated by Prompt 7
2. **Test Cases**: Paste the complete test case content generated by Prompt 6

### Output Expectations
1. **BDD Execution Framework**: Behave test runners with appropriate environment settings and dependencies
2. **Automated Execution Scripts**: Scripts to run BDD scenarios automatically and collect results, stored in `tests/bdd/automation/` directory
3. **Result Aggregation Framework**: Frameworks to aggregate and analyze BDD execution results across multiple runs
4. **BDD Execution Reports**: Detailed BDD execution reports with scenario-level analysis and trend tracking
5. **CI/CD Integration**: CI/CD pipeline configurations for BDD scenario execution
6. **Process File**: `PROMPT11-OUTPUT.md` stored in `governance/` directory

### Usage Steps
1. **Preparation**:
   - Gather BDD scenarios from Prompt 7 output
   - Gather test cases from Prompt 6 output
   - Configure test environment and dependencies

2. **Execution**:
   - Run Prompt 11 with prepared inputs
   - Review generated BDD execution framework
   - Validate automated execution scripts
   - Verify result aggregation framework
   - Test CI/CD integration

3. **Review**:
   - Use `governance/reviews/prompt11-review.md` for review tracking
   - Apply `governance/reviews/test-execution-review-template.md` for structured feedback
   - Calculate confidence levels using `governance/reviews/confidence-assessment.md`
   - Perform failure analysis if needed using `governance/reviews/failure-analysis-template.md`

4. **Implementation**:
   - Set up BDD execution framework
   - Configure automated execution scripts
   - Implement result aggregation framework
   - Set up CI/CD pipeline
   - Run BDD scenarios

5. **Verification**:
   - Verify BDD scenario execution
   - Validate results collection
   - Confirm aggregation framework functionality
   - Test CI/CD pipeline integration
   - Review execution reports

### User Type Templates
- **Type A (BA)**: Business-focused BDD execution templates with business rule verification

## Best Practices

### For Prompt 10
1. Always validate test execution plans before implementation
2. Ensure results validation frameworks are robust and objective
3. Verify coverage analysis completeness
4. Test automation scripts in isolation before full implementation
5. Document all execution results thoroughly

### For Prompt 11
1. Always test BDD execution framework with sample scenarios
2. Verify automated execution scripts handle edge cases
3. Ensure result aggregation framework captures all necessary metrics
4. Test CI/CD integration in a staging environment
5. Review BDD execution reports for actionable insights

### Common to Both
1. Use appropriate user type templates
2. Follow review workflow strictly
3. Calculate confidence levels objectively
4. Perform failure analysis when needed
5. Maintain complete execution history

## Troubleshooting

### Common Issues
1. **Execution Environment Issues**: Verify test environment setup and dependencies
2. **Validation Framework Failures**: Check validation logic and expected outcomes
3. **Coverage Analysis Gaps**: Review coverage metrics and identify missing areas
4. **Automation Script Errors**: Debug scripts and test in isolation
5. **CI/CD Integration Failures**: Verify pipeline configurations and dependencies
