## Template Content Standards

### Common Template Structure

All templates should include:

1. **Header Section**
   - Template ID
   - Creation date
   - Associated prompt
   - Version number

2. **Review Information**
   - Review ID
   - Review date
   - Reviewer name
   - Review stage
   - Status

3. **Review Criteria**
   - Rule alignment checks
   - Completeness checks
   - Quality checks
   - Traceability checks

4. **Feedback Section**
   - Strengths
   - Areas for improvement
   - Specific comments
   - Action items

5. **Approval Section**
   - Decision
   - Rationale
   - Next steps
   - Signatures

### Template Creation Guidelines

When updating prompts to include template creation:

1. **Include Complete Template Structure**
   - All sections must be defined
   - All fields must be documented
   - Example values should be provided

2. **Specify File Location**
   - Exact directory path
   - File naming convention
   - File format (Markdown)

3. **Define Template Usage**
   - When to use the template
   - Who should use it
   - How to fill it out

4. **Ensure Consistency**
   - Use consistent terminology
   - Follow naming conventions
   - Maintain formatting standards

## Implementation Notes

### Template Inheritance

Some templates can be shared across prompts:
- `feedback-template.md` - Used by Prompt 6 and 7
- `confidence-assessment.md` - Used by Prompt 6 and 7
- `failure-analysis-template.md` - Used by Prompt 6 and 7

### Template Customization

Each prompt should customize the template for its specific context:
- Update section headers to match prompt output
- Add prompt-specific review criteria
- Include prompt-specific metadata fields

### Quality Assurance

When implementing template creation in prompts:
1. Ensure all templates are complete and usable
2. Verify template locations are correct
3. Test template generation with sample inputs
4. Validate template consistency across prompts
